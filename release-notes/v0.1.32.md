# Release v0.1.32

<img src="https://github.com/kopecmaciej/vi-mongo/blob/master/assets/logo/no-background.svg" align="center"/>

## What's New

### Features

- **Regex Pattern Support**: Added comprehensive support for MongoDB regex patterns and literals in queries [#97](https://github.com/kopecmaciej/vi-mongo/issues/97)
  - Support for regex literal syntax (e.g., `/pattern/flags`)
  - Support for MongoDB helper functions: `ISODate()`, `NumberInt()`, `NumberLong()`, `NumberDecimal()`
  - Improved array handling in regex context
  - Added extensive utility functions for regex pattern processing

### Bug Fixes

- **Copy Cell Value**: Fixed issue where copying a cell value in table view would copy the truncated display text instead of the full value
  - Now retrieves the actual document field value when copying
  - Example: Copying a cell showing { "object": "full_long_val..., now copies the full { "object": "full_long_value" }

### Improvements

- **Error Handling**: Added panic recovery mechanism to display properly formatted error messages to users, improving the debugging experience and preventing abrupt application crashes
- **Testing**: Added comprehensive test coverage for command handling and regex pattern processing
- **Code Refactoring**: Improved state management by moving field value retrieval logic from UI components to state package
  - Added `GetValueByColumn` method to `CollectionState` for better separation of concerns
  - Reduced code duplication in content component (~30 lines removed)

### Technical Changes

- **Code Organization**: Refactored regex handling into dedicated utility module (`internal/util/regexp.go`)
- **Parser Enhancement**: Updated MongoDB query parser to handle regex patterns and MongoDB helper functions
