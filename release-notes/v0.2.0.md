# Release v0.2.0

<img src="https://github.com/kopecmaciej/vi-mongo/blob/master/assets/logo/no-background.svg" align="center"/>

## What's New

> **Breaking change:** `keybindings.json` is no longer supported. Your file will be automatically migrated to `keybindings.yaml` on first startup. A backup is saved as `keybindings.json.bak`.

### Features

- **Aggregation Pipeline Builder**: A dedicated tab for building and running MongoDB aggregation pipelines directly from the TUI. Stages are composed interactively and the full pipeline is executed against the selected collection.
- **Live Stage Preview**: Preview pipeline output before running the full aggregation — limited to 5 documents per stage so you can iterate quickly.
- **Persistent Stages**: Pipeline stages are preserved when switching between collections within the same session.
- **In-App Changelog**: A changelog modal is shown on first startup after an upgrade, so you always know what changed.
- **Dot Notation in Headers**: Table column headers now support dot notation, correctly resolving nested field paths.

### Bug Fixes

- **Ctrl + Lowercase Key Bindings**: Fixed resolution of `Ctrl+<lowercase letter>` bindings that were not triggering correctly across all components.
- **Paste Cleanup**: Pasting content into the aggregation editor now strips unnecessary surrounding braces when appropriate.
- **Delete Modal**: Fixed the delete confirmation modal behavior in the aggregation view.

### Improvements

- **Keybindings Migration to YAML**: `keybindings.json` is automatically migrated to `keybindings.yaml` on startup. No manual action required. YAML was chosen over JSON for a reason — the keybindings file groups keys by component, and with JSON's required quoting and lack of comments, scanning through dozens of entries to find a specific binding was tedious. YAML's cleaner syntax, support for comments, and visual indentation make it much easier to scan, spot a component section at a glance, and edit a single key without fighting brackets and trailing commas.
- **Unified Delete & Refresh Keys**: Delete and refresh keybindings are now consistent across all components.
- **Dot Notation Support**: Added dot notation support for parsing nested document fields in table column headers.
