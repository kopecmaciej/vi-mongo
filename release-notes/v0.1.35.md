# Release v0.1.35

<img src="https://github.com/kopecmaciej/vi-mongo/blob/master/assets/logo/no-background.svg" align="center"/>

## What's New

### Features

- **Expandable Header**: Added a collapsible header that shows all available keybindings for the currently focused component. Press `t` to toggle between collapsed and expanded states. The expanded view uses a responsive multi-column grid layout based on terminal width.
- **Esc to Cancel Connection Form**: Added `Escape` key support to dismiss the connection form and return to the connection list.
- **Environment Variable URI Support**: Connection URIs now support environment variable references (e.g. `$MONGODB_URI`). The value is stored as-is in the config and expanded at connection time, making it safe to version-control the config file without embedding credentials.

### Bug Fixes

- **Connection Rename Sync**: Fixed an issue where renaming the currently active connection would not update the `currentConnection` field, causing a mismatch after save.

### Improvements

- **Secure Config File Permissions**: Config file is now written with mode `0600` (owner read/write only) instead of `0644`, preventing other users on the system from reading connection credentials.
- **Connection Form UX**: Improved the connection form key hint text to clearly show both save and exit keys (`<key> to save, Esc to exit`). Simplified cancel/edit flow by merging redundant cancel functions.
- **Header Cleanup**: Removed database and collection info from the sys-info header row — it now shows only host and port for a cleaner, less noisy header.
