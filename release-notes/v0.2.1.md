# Release v0.2.1

<img src="https://github.com/kopecmaciej/vi-mongo/blob/master/assets/logo/no-background.svg" align="center"/>

## What's New

### Features

- **In-App Keybinding Editor**: Keybindings can now be remapped directly from the help page. It can be done by navigating to any binding and editing it in place using keys or letters (runes).
- **Arrow Key Navigation**: Full support for `up`, `down`, `left`, and `right` keybindings across all components — navigation areas now use a unified set of navigation keys.
- **Jump to Next Marker**: In the editor, pressing the next-marker key moves the cursor to the next placeholder marker, making template-based editing faster.
- **Type Security in Editor**: When editing a document in editor mode, values are now type-safe — the editor preserves BSON types rather than coercing everything to strings.
- **Index Key Bindings**: Configurable keybindings for adding indexes are now exposed in the help page and keybindings config.

### Bug Fixes

- **Style Fixes**: Corrected styling inconsistencies in the aggregation builder and database tree components.
- **Help Page Edit Hint**: Fixed the edit hint message shown in the help page.
- **Duplicate ctrl+r Binding**: Removed a duplicated `ctrl+r` key assignment that caused ambiguous behavior.

### Improvements

- **Redesigned Help Page**: Keybindings are now grouped per-component with clearer visual hierarchy, making it easier to discover and edit bindings.
- **Unified Navigation Keys**: A single set of navigation keys (`up`, `down`, `left`, `right`) is now shared across all components and pages, eliminating duplication.
- **Input Modal Key Cleanup**: Removed unused keybindings from the input modal; view-modal navigation is cleaner and more consistent.
